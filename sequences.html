The following sequence diagrams illustrate the behaviour of the Connected Device in particular scenarios including the expected ordering of interactions with the Mobile Device.

Note that Device Identification Service and Device Time Service requests can occur at any point during the connection and do not impact the behaviour depicted below.

Successful pairing and key exchange#

The diagram below shows the expected interactions with the Mobile Device when the Connected Device is performing pairing. Note that the read request on the Connected Device Public Key characteristic and the write request on the Mobile Device Public Key characteristic can be performed in either order.

Successful key exchange, already paired#

If the Mobile Device and the Connected Device are already paired, the pairing step can be skipped because the connection will already be encrypted, as shown in the diagram below.

Disallowed requests before pairing#

The diagram below shows the behaviour of the Connected Device when the requests are received before pairing has succeeded. Any one of these InsufficientEncryption responses will trigger the operating system on the Mobile Device to initiate pairing.

Successful Data Synchronisation#

The diagram below shows the expected interactions with the Mobile Device when the Connected Device is in data sync mode. Note that an Exchange MTU Request may be sent at any point during the connection and the next Record Response will utilise the new ATT MTU size.
