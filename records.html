Record format#

Dose Records and Error Records are five byte (forty bit), bit-packed records containing dose or device error information.

The Dose Time is stored as minutes since First Use Time so this value must be added to the First Use Time to get the Epoch Time of a dose: Epoch Time (dose) = First Use Time + (Dose Time x 60).

The Check Value is the output of a CRC calculation on a combination of the Record Number and the first 4 bytes of the record. The two byte Record Number is prepended to the record, creating a 6 byte input to the Check Value calculation.

Bit(s) Field Description
0 to 20 DOSE TIME The time of the dose, in minutes since FIRST USE TIME or may be an error code, see below.
21 to 27 REPORTED DOSE The dose reported by the Connected Device, may be an error code, see below.
28 BUTTON TIMEOUT FLAG 1 if the DOSE BUTTON was held for at least BUTTON_TIMEOUT, 0 otherwise.
29 VERY FAST DOSE FLAG 1 if a very fast dose was detected, 0 otherwise.
30 LOW VOLTAGE FLAG 1 for battery voltage below LOW_VOLTAGE, 0 otherwise.
31 FAST DOSE FLAG 1 if a FAST DOSE was detected, 0 otherwise.
32 to 39 CHECK VALUE The CHECK VALUE for the record (bytes 0 to 1 of the DOSE NUMBER followed by bytes 0 to 3 of the record).

Check value calculations#

The Dose Record, Error Record and Extended Dose Record Check Values are the output of a CRC calculation on the specified preceding bytes with the following “Rocksoft” CRC parameters:

  • Width: 8
  • Poly: 0x97
  • Init: 0xFF
  • RefIn: False
  • RefOut: False
  • XorOut: 0x00
  • Check: 0x64 (Output of a ‘verify’ test on the 9 byte ASCII string “123456789”, excluding the quotes)

The CRC parameters are chosen based upon work by Koopman et al. (see here). The parameters permit error detection to be performed by the CRC where up to 4 bits are changed in a maximum input length of 119 bits. This covers the use of the algorithm in calculating Check Values for Dose Records, Error Records and Extended Dose Records.

Identifying errors#

The following flags indicate that the Dose Record may contain unreliable dose data:

  • Button Timeout Flag: further units may have been dispensed after dose measurement stopped.
  • Fast Dose Flag: the dose may have been an air shot. (see here)
  • Very Fast Dose Flag: units may have been missed due to the dispense speed. (see here)
  • Low Voltage Flag: the battery voltage following dose measurement was less than or equal to LOW_VOLTAGE.

The following should be checked to identify other potential errors:

  • The Reported Dose of the current record – it could indicate a Dose Error or a Device Error.
  • The Dose Time of the current record – it could indicate a Time Error.
  • The Reported Dose of the previous record – it could indicate a Device Error.
  • The Dose Time of the previous record – it could indicate that the Dose Times are out of order.

Time errors#

The following Time Errors can be reported in the Dose Time field of a record.

DOSE TIME value Description
0x1FFFF0 Communications failure with the RTC.
0x1FFFF1 RTC has reset.
0x1FFFF2 RTC oscillator has failed.
0x1FFFF3 Incorrect configuration of the RTC.
0x1FFFF4 The RTC holds an invalid date or time e.g. (49/61/2019 70:98:77).
0x1FFFF5 FIRST USE TIME is before production date set in the UDI or after 03:14:07 on 19th January 2038.
03:14:07 on 19th January 2038 represents the latest EPOCH TIME that is supported by the Connected Device.
0X1FFFF6 Device time is before the production date set in the UDI or after 03:14:07 on 19th January 2038.
03:14:07 on 19th January 2038 represents the latest EPOCH TIME that is supported by the Connected Device.
0X1FFFF7 Device time is before FIRST USE TIME.
0x1FFFF8 The calculated DOSE TIME is larger than the maximum supported value 0x1FFFEF (may indicate the device has been in use for more than 3.99 years)
0x1FFFFF Internal consistency check failed when calculating DOSE TIME.

Info

Where more than one TIME ERROR occurs the one with the lowest value is reported in the record.

Dose errors#

The following Dose Errors can be reported in the Reported Dose field of a record.

REPORTED DOSE value Description
81 The MEASURED DOSE was 81 Units.
Technically feasible in some mechanical tolerance situations prior to clutch re-engagement, e.g. if a BUTTON TIMEOUT occurs and dose measurement stops before the mechanism reverses as the clutch re-engages.
82 The MEASURED DOSE was larger than 81 Units.
This would indicate a mechanical device failure and should not occur, even in a mechanically abused device.
83 The MEASURED DOSE was -1.
Technically feasible in some mechanical tolerance situations if the mechanical mechanism is mildly abused, most probably with a dialled dose of zero.
84 The MEASURED DOSE was <-2 without two consecutive Gray code transitions decrementing the MEASURED DOSE.
This would indicate a software failure.
85 Two consecutive Gray code transitions decremented the MEASURED DOSE.
This would indicate a mechanical device failure due to abuse, most probably with a non-zero dialled dose.
86 The MEASURED DOSE was -2.
This would indicate a mechanical device failure due to abuse, most probably with a dialled dose of zero.
87 Sensor controller initialisation error: the sensor controller failed to start so no dose measurement could take place.
88 Sensor controller run time error: the sensor controller started but failed to terminate normally and return dose data.
95 Internal consistency check failed when determining reported dose value.

Info

Where more than one DOSE ERROR occurs the one with the highest value is reported.

Device errors#

When a Device Error is detected the Connected Device generates an Error Record which has the same format as a Dose Record but the Reported Dose field contains information relating to the detected Device Error.

Error Records may have an associated Dose Record in which case the Error Record will be stored immediately preceding the associated Dose Record and will share the same Dose Time.

An Error Record may be distinguished from a Dose Record by the value of the Reported Dose field.

REPORTED DOSE value Description
96 The battery voltage was measured to be below VERY_LOW_VOLTAGE.
This may indicate the coin cell is close to end of life but may also be due to a low temperature (from say storing in a refrigerator) depressing the voltage, which may recover when the temperature of the device increases.
97 Reserved.
98 Microcontroller reset. Reason: unknown/other.
An 'unknown/other' microcontroller reset is reported when the reason for the reset cannot be determined by the microcontroller.
99 Microcontroller reset. Reason: the microcontroller has been reset via the TCK JTAG input pin, which may indicate a hardware fault or potential tampering.
100 Microcontroller reset. Reason: the microcontroller has been reset via the hardware pin reset input, which may indicate a hardware fault or potential tampering.
101 Microcontroller reset. Reason: the microcontroller watchdog has triggered a microcontroller reset.
102 Microcontroller reset. Reason: the microcontroller has entered a CPU lockup state due to a software failure.
103 Microcontroller reset. Reason: the crystal oscillator has ceased running.
104 Microcontroller reset. Reason: the battery voltage has momentarily dropped below BROWN_OUT_VOLTAGE.
105 Microcontroller reset. Reason: the power to the microcontroller has been cycled.
127 Internal consistency check failed when determining the DEVICE ERROR.

Info

Where more than one DEVICE ERROR occurs the one with the highest value is reported.
