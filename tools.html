AllStar® Connect Device Simulator#

Intended Use:

The AllStar® Connect Device Simulator is a test unit intended to support software design integration and verification activities of any 3rd party app developer who is integrating the AllStar® Connect by simulating all error states that cannot be generated through normal use of the AllStar® Connect pen.

The system comprises a Bluetooth communicating test unit controlled by text commands over a virtual serial interface running on a computer connected via USB cable. This command system is used for all interaction with the simulator. Besides configuration there are a series of commands available to simulate pressing the button and achieving any of the data communication protocol behaviour (not including Bluetooth stack level control of errors). This allows to trigger pairing, syncing and dosing, and also having control over the REPORTED DOSE, DOSE TIME, Battery voltage, Microprocessor temperature, flags, and CRC value. This means one can create any dose error, device error, time error and create a CRC failure.

A Device Simulator with detailed documentation can be provided.

Test App#

A working MVP App can be provided together with detailed documentation.

The App is Android based and supports basic functionality such as scanning for devices, pairing devices and synchronizing dose data.

Bluetooth Sniffer#

We are using a Frontline Sodera LE Wideband Buetooth Protocol Analyzer for monitoring our Bluetooth connections. The main requirement is that it supports Bluetooth 4.2.

Link to website here.

The software provided by Frontline is quite good, you can see all the different packets through the layers of the Bluetooth stack which is incredibly useful for determining whether device A or phone B is misbehaving!

However, we have found that it cannot decrypt messages on a link encrypted using Secure Connections Pairing which is used for ASC. It is possible to enable “debug keys” in our ASC software but this requires a special build so any formal testing of the connection after pairing cannot rely on the sniffer.
